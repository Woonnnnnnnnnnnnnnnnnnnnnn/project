TCP(Tranmission Control Protocol)
โปรโตคอลควบคุมการส่งข้อมูลแบบมุ่งเน้นการเชื่อมต่อ

TCP คืออะไร?

การอธิบายพื้นฐานเกี่ยวกับโปรโตคอล TCP ในการส่งข้อมูลที่เชื่อถือได้

TCP (Transmission Control Protocol) เป็นโปรโตคอลที่ทำงานในชั้น Transport Layer ของโมเดล OSI และถูกออกแบบมาเพื่อให้การส่งข้อมูลที่มีความน่าเชื่อถือระหว่างสองเครื่องคอมพิวเตอร์ในเครือข่าย โดย TCP รับประกันว่าข้อมูลจะถูกส่งถึงปลายทางอย่างสมบูรณ์และถูกต้องตามลำดับ

ภาพแสดงภาพรวมการทำงานของ OSI

การทำงานหลักของ TCP

การเชื่อมต่อแบบ Three-Way Handshake และวิธีการควบคุมการไหลของข้อมูล

Three-Way Handshake: เป็นขั้นตอนการเริ่มต้นการเชื่อมต่อ ซึ่งประกอบด้วยการส่งแพ็กเก็ต 3 ครั้ง: SYN (Synchronize) จากต้นทาง, SYN-ACK (Synchronize-Acknowledge) จากปลายทาง, และ ACK (Acknowledge) จากต้นทางอีกครั้งเพื่อยืนยันการเชื่อมต่อ

การส่งข้อมูล: เมื่อการเชื่อมต่อสำเร็จแล้ว, TCP จะทำการแบ่งข้อมูลออกเป็นส่วนย่อยๆ ที่เรียกว่า Segment และส่งไปตามลำดับ โดยใช้ Sequence Numbers เพื่อติดตามลำดับของข้อมูล

การยืนยันการรับข้อมูล (Acknowledgement): เมื่อปลายทางได้รับ Segment แล้วจะส่ง ACK กลับมายังต้นทางเพื่อยืนยันว่าข้อมูลได้รับเรียบร้อยแล้ว หากไม่มีการส่ง ACK กลับมา ต้นทางจะทำการส่งข้อมูลใหม่

การควบคุมการไหล (Flow Control): TCP ใช้เทคนิคที่เรียกว่า Sliding Window เพื่อควบคุมปริมาณข้อมูลที่สามารถส่งได้ในแต่ละช่วงเวลา ป้องกันไม่ให้ปลายทางได้รับข้อมูลมากเกินไปจนเกิดปัญหา

การจัดการความแออัด (Congestion Control): TCP ใช้เทคนิคต่างๆ เช่น **Slow Start** และ **Congestion Avoidance** เพื่อควบคุมปริมาณข้อมูลที่ส่งในช่วงที่มีการแออัดในเครือข่าย เพื่อลดการสูญเสียข้อมูล

ข้อเสียของTCP

ข้อเสียของ TCP มีอะไรบ้าง

  • ความซับซ้อน (Complexity): TCP เป็นโปรโตคอลที่ซับซ้อนมาก เพราะต้องจัดการหลายอย่างพร้อมกัน ทั้งการสร้างและยุติการเชื่อมต่อ, การตรวจสอบข้อผิดพลาดของข้อมูล, และการรับรองว่าข้อมูลถูกส่งถึงปลายทาง ทำให้ต้องใช้ทรัพยากรทั้งหน่วยความจำและ CPU เยอะกว่าโปรโตคอลที่เรียบง่ายกว่าอย่าง UDP
  • ประสิทธิภาพต่ำในบางกรณี (Lower Performance): เนื่องจากต้องใช้กระบวนการที่ซับซ้อนในการส่งข้อมูล ทำให้ TCP มีความเร็วที่ช้ากว่า UDP โดยเฉพาะในแอปพลิเคชันที่ต้องการความเร็วสูงและยอมให้ข้อมูลขาดหายได้เล็กน้อย เช่น การสตรีมวิดีโอ, เกมออนไลน์ หรือการโทรศัพท์ผ่านอินเทอร์เน็ต (VoIP)
  • ค่าใช้จ่ายสูง (Overhead): แต่ละแพ็กเก็ตของ TCP จะมีส่วนหัวข้อมูล (Header) ขนาดใหญ่ถึง 20-60 ไบต์ ซึ่งใหญ่กว่าส่วนหัวของ UDP ที่มีขนาดเพียง 8 ไบต์เท่านั้น ทำให้ต้องใช้แบนด์วิธในการส่งข้อมูลเพิ่มขึ้น และทำให้ข้อมูลจริงที่ส่งมีสัดส่วนน้อยลงเมื่อเทียบกับข้อมูลทั้งหมด
  • ความหน่วง (Latency): กระบวนการทำงานของ TCP ที่ต้องรอการตอบรับ (Acknowledgement) จากปลายทางก่อนจะส่งแพ็กเก็ตชุดถัดไป ทำให้เกิดความหน่วง หรือดีเลย์ขึ้นมา ซึ่งเป็นปัญหาใหญ่สำหรับแอปพลิเคชันที่ต้องการตอบสนองแบบเรียลไทม์
  • ไม่เหมาะกับการส่งข้อมูลแบบกระจาย (Multicast): TCP ถูกออกแบบมาเพื่อการส่งข้อมูลแบบหนึ่งต่อหนึ่ง (Unicast) ทำให้ไม่รองรับการส่งข้อมูลไปยังหลายๆ เครื่องพร้อมกัน (Multicast) ซึ่งถ้าต้องการส่งข้อมูลให้หลายเครื่องพร้อมกันต้องใช้โปรโตคอลอื่น หรือต้องใช้เทคนิคที่ซับซ้อนเข้ามาช่วยครับ

ข้อดีของ TCP

ข้อดีที่ทำให้ TCP เป็นโปรโตคอลที่นิยมใช้ในหลายระบบ

  • Reliability: การส่งข้อมูลที่มีความน่าเชื่อถือสูง เหมาะสำหรับแอปพลิเคชันที่ข้อมูลห้ามสูญหาย เช่น การโอนไฟล์หรือการท่องเว็บ
  • Error Checking: มีการตรวจสอบข้อผิดพลาดและส่งข้อมูลซ้ำหากจำเป็น ทำให้มั่นใจได้ว่าข้อมูลที่ได้รับถูกต้อง
  • Efficiency: แม้จะมีการทำงานที่ซับซ้อน แต่ก็มีประสิทธิภาพในการจัดการการส่งข้อมูลจำนวนมาก
  • Efficiency: การควบคุมการไหล (Flow Control) TCP ปรับอัตราการส่งข้อมูลให้เหมาะสมกับความสามารถในการประมวลผลของผู้รับ เพื่อป้องกันไม่ให้ผู้รับถูกท่วมไปด้วยข้อมูลที่มากเกินไปจนไม่สามารถจัดการได้
  • Efficiency: การควบคุมความแออัด (Congestion Control) TCP มีกลไกในการตรวจจับและจัดการกับความแออัดในเครือข่าย ทำให้การส่งข้อมูลเป็นไปอย่างมีเสถียรภาพ และลดโอกาสเกิดปัญหาข้อมูลสูญหายเนื่องจากเครือข่ายทำงานหนักเกินไป

ความแตกต่างระหว่าง TCP และ UDP

ควรใช้แต่ละอย่างเมื่อใด?

TCP: ให้ความสำคัญกับความน่าเชื่อถือและความปลอดภัยของข้อมูล ทำให้ข้อมูลถูกส่งอย่างครบถ้วน แม้จะต้องใช้เวลามากขึ้น

UDP: เป็นโปรโตคอลที่เร็วกว่า แต่ไม่มีการรับประกันความน่าเชื่อถือของข้อมูล เหมาะสำหรับการใช้งานที่ต้องการความเร็วสูง เช่น การสตรีมวิดีโอ การโทรศัพท์ผ่านอินเทอร์เน็ต (VoIP) หรือการเล่นเกมออนไลน์

  • การเชื่อมต่อ: TCP ต้องการการสร้างการเชื่อมต่อก่อน (Three-Way Handshake) ในขณะที่ UDP สามารถส่งข้อมูลได้ทันทีโดยไม่ต้องมีการสร้างการเชื่อมต่อ
  • ความน่าเชื่อถือ: TCP รับประกันความถูกต้องของข้อมูลและมีการส่งข้อมูลซ้ำหากสูญหาย, UDP ไม่มีการรับประกันนี้
  • ความเร็ว: UDP มีความเร็วสูงกว่าเพราะไม่มีการตรวจสอบหรือการควบคุมข้อผิดพลาด

ภาพเปรียบเทียบความแตกต่างระหว่าง TCP และ UDP
